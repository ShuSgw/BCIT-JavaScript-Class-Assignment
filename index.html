Variables

Variables are containers that can store data values, and you can use them whenever you need. However, we need to consider that there are different types and scopes. Also, variable can store another variable or themselves.

Declaring Variable

The way to create a variable is a keyword “var”. (There are other keywords but I will explain later.) And you have to declare a name you like. However, double byte languages such as Japanese, spaces, comma, question mark, quotation, numbers as the first letter, and reserved words are not allowed.

						
	<script>
	   var name = 0;
	   var name = "Hello";

	   //they are not allowed to use↓
	   var 日本語 = "Japanese";
	   var ze ro = 0;
	   var hello, world = "text";
	   var "hello" = "hello";
	   var 1hello = "text";
	   var function = "text";
	</script>

There are three different types of variables. When you delete variables you should consider how you will use them.
A variable with keyword:

  • var : The variable is readable and editable
  • const : The variable is readable only
  • let : The variable is only work inside of a block

						
	<script>
		const Csample = 0:
		var Vsample = 0;

		Csample = 1:

		console.log(Csample);//1

		function f() {

Scope

There are 3 different types of scope in Javascript depending on where variables are decread and how you access them.

Global scope

When variables are declared outside of functions or blocks, these variables are global variables. Also, you can use global variable whenever you want.

Local scope

When variables are declared inside of a function or as a parameter for the function, these variable are local variables. Also, you can only use local variables within the function. If you use these local variables declared within a function outside of the function, it occurs some errors.

Block scope

When variables are declared within a code block, such as if or for statements with keyword “let”, these variable become brock variables. 

						
	<script>

	//Global
	var x = 1;
	function(){
		console.log(x);//1
	}

	//local
	function(){
		var x = 1;
		console.log(x);//1
	}

	//block
	function f() {
		  let x = 1;
		  console.log(x);
		  {
		    let x = 2;
		    console.log(x);
		  }
		  console.log(x);
		}
	f()
	//1
	//2
	//1

	</script>

Function

Set of statements for performing some tasks, such as calucration. You can name it whatever you want, but it only works when it is called.

Function invocation

After you define a function, it is just a function, so you need to invoke with some actions or name you decide, which is function invocation.

						
	<script>
	function(){
		console.log("hello");
	}

	function();
	//hello

	</script>

Function parameters

You can use a function you define many times, and pass some values to the function. Then, the function performs whatever you define, and has different values.

						
	<script>
	function hello(a){
		console.log(a);
	}

	hello(1);
	//1
	hello(2);
	//2
	hello("hello");
	//"hello"

	</script>

Conditional Structures

When you need to separate tasks depending on some situations, the conditional structures should be used.There are two ways to create conditional structures in Javascript.

If/else

A condition you define is evaluated by “if” statement, and if a situation matches the condition, it executes a block you define. Also, you can create another different block from the true part. If the situation does not match the condition you define, “if else” statement executes another block as false. Moreover, you can add more conditions with “else if”.

						
	<script>
		var sample = 1;

		//if
		if(sample > 0){
			console.log("sampple is more than 0");
		}

		//if else
		var score = 90;
			if (score >= 80) {
				document.write(“Pass!!”);
			} else {
				document.write(“Fail!!”);
			}

		//elase
		var score = 90;
			if (score >= 80) {
				document.write(“Great”);
			} else if (score >= 50) {
				document.write(“Pass”);
			} else{
				document.write(Fail);
			}
	</script>

Swich

When you have many conditions and bloks, you should use switch instead of using if statement. The value of the expression you define is compared with each case, then if the expression matches a case, the case’s block is executed.

Break

“Break” keyword can stop after a case matches to the expression, so each case has to have the keyword at the end.

Default

You can define a exception that each case does not match the expression you define to create “default”(case).

						
	<script>
		var weather = 'Sun';
		 
		 
		switch( weather ) {

				case default:
				        console.log('Not sure yet');
				        break;

		    case 'Sun':
		        console.log('It is sunny today');
		        break;
		 
		 
		    case 'Cloud':
		        console.log('It is cloudy today');
		        break;
		 
		 
		    case 'Rain':
		        console.log('It is rainy today');
		        break;
		}
	</sript>
